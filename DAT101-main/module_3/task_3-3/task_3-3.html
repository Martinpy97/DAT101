1. What is the difference between invoking and creating a function?

    2. Når man oppretter en funksjon, så definerer du den. eksempel "function eksempel() { PrintOut("Hei");}
    Når man kaller på den så bruker man den ved å skrive "eksempel();"

  2. Explain the syntax regarding semicolons and code blocks.

    3. Semikolon brukes i JavaScript for å markere slutten på en setning eller en kommando,
    som for eksempel når du tildeler verdier eller utfører beregninger. Eksempel: "let x = 5;"

    Kodeblokker brukes for å gruppere flere linjer med kode sammen, vanligvis innenfor funksjoner, løkker eller betingelser.
    De bruker "curly brackets" { }. Eksempel:"if (x > 0) { PrintOut("positivt tall"); }"

  3. What happens when we try to store values from a function when it doesn't return any?

    4. Når en funksjon ikke returnerer noe, betyr det at funksjonen ikke gir noen verdi tilbake til koden som kaller på den.
    Hvis du prøver å lagre resultatet av en funksjon som ikke returnerer noe, vil variabelen som lagrer verdien bli undefined.

  4. How do we separate parameters from each other when creating a function?

    5. Når du lager en funksjon og definerer dens parametere, separeres de med komma.
    Parametrene er plassholdere for verdier som vil bli sendt til funksjonen når den kalles.

  5. Explain how parameter names are related during the invocation and creation of a function.

    6. Når du lager en funksjon, definerer du parameternavnene. Når du kaller på funksjonen, sender du verdier (argumenter) til funksjonen.
    Parametrene i funksjonen vil da bruke de verdiene som er sendt som argumenter.

  6. Explain why a parameter is undefined when a function call is missing a parameter.

    7. Når en funksjon er kalt uten at en parameter er sendt med, blir verdien for den manglende parameteren automatisk satt til undefined.
    Dette skjer fordi funksjonen forventer et argument, men ikke får det.

  7. Explain what happens when we use recursive functions.

    8. En rekursiv funksjon er en funksjon som kaller seg selv for å løse et problem. Dette skjer gjentatte ganger til den når et base case, som er en betingelse for at funksjonen skal stoppe.
    Hvis det ikke finnes et base case, vil funksjonen kalle seg selv uendelig og som vil gi feil.
